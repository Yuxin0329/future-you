# Future You 启动指南

## 前置条件检查

### 1. 检查 Node.js 版本

```bash
node -v
# 应该显示 v18.0.0 或更高版本
```

如果未安装或版本过低，请参考 [部署说明.md](./部署说明.md) 安装 Node.js。

### 2. 检查 MongoDB 状态

#### 方法一：使用 systemd 服务（推荐）

```bash
# 检查 MongoDB 服务状态
sudo systemctl status mongod

# 如果未运行，启动服务
sudo systemctl start mongod

# 设置开机自启
sudo systemctl enable mongod
```

#### 方法二：使用 Docker（备选方案）

如果没有系统权限安装 MongoDB，可以使用 Docker：

```bash
# 拉取 MongoDB 镜像
docker pull mongo:6.0

# 运行 MongoDB 容器
docker run -d \
  --name future-you-mongodb \
  -p 27017:27017 \
  -v mongodb_data:/data/db \
  mongo:6.0

# 查看容器状态
docker ps

# 停止容器
docker stop future-you-mongodb

# 启动容器
docker start future-you-mongodb
```

#### 方法三：使用 MongoDB Atlas（云端）

1. 访问 [MongoDB Atlas](https://www.mongodb.com/atlas)
2. 注册并创建免费集群
3. 获取连接字符串
4. 修改 `server/.env` 中的 `MONGODB_URI`

```env
MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/future-you
```

## 启动步骤

### 1. 配置环境变量

检查 `server/.env` 文件是否存在且配置正确：

```env
PORT=3000
NODE_ENV=development

# MongoDB配置
MONGODB_URI=mongodb://localhost:27017/future-you

# JWT密钥
JWT_SECRET=future-you-secret-key-2024-very-secure
JWT_EXPIRES_IN=7d

# CORS配置
CORS_ORIGIN=http://localhost:5173
```

### 2. 启动后端服务器

```bash
cd server

# 开发模式（支持热重载）
npm run dev

# 生产模式
npm start
```

后端服务将在 `http://localhost:3000` 启动

如果看到以下信息，说明启动成功：

```
✅ MongoDB connected successfully
🚀 Future You Server is running on port 3000
📧 API: http://localhost:3000/api
💗 Environment: development
```

### 3. 启动前端开发服务器

打开一个新的终端窗口：

```bash
cd client

# 开发模式
npm run dev
```

前端服务将在 `http://localhost:5173` 启动

如果看到以下信息，说明启动成功：

```
  VITE v5.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
```

### 4. 访问应用

在浏览器中打开：`http://localhost:5173`

## 开发工作流

### 同时启动前后端

#### 方法一：使用两个终端窗口

**终端 1 - 后端：**
```bash
cd server
npm run dev
```

**终端 2 - 前端：**
```bash
cd client
npm run dev
```

#### 方法二：使用一个终端和 & 符号

```bash
# 在后台启动后端
cd server && npm run dev &

# 启动前端
cd ../client && npm run dev
```

#### 方法三：使用 concurrently（推荐）

安装 concurrently：

```bash
npm install -g concurrently
```

在项目根目录创建启动脚本：

```bash
# 在根目录运行
concurrently "cd server && npm run dev" "cd client && npm run dev"
```

## 停止服务

### 停止前端开发服务器

在运行前端的终端窗口中，按 `Ctrl + C`

### 停止后端服务器

在运行后端的终端窗口中，按 `Ctrl + C`

### 停止 MongoDB（如果使用 Docker）

```bash
docker stop future-you-mongodb
```

## 常见问题

### Q: 端口 3000 被占用

**A:** 修改 `server/.env` 中的 `PORT` 配置：

```env
PORT=3001
```

然后重启后端服务。

### Q: 端口 5173 被占用

**A:** 在 `client/vite.config.js` 中修改端口配置：

```javascript
export default defineConfig({
  // ...
  server: {
    port: 5174,  // 改为其他端口
    // ...
  }
})
```

### Q: MongoDB 连接失败

**A:** 检查 MongoDB 是否运行：

```bash
# 如果使用 systemd
sudo systemctl status mongod

# 如果使用 Docker
docker ps | grep mongo
```

如果未运行，请参考上文的方法启动 MongoDB。

### Q: 前端请求后端 API 失败（CORS 错误）

**A:** 检查 `server/.env` 中的 `CORS_ORIGIN` 配置：

```env
CORS_ORIGIN=http://localhost:5173
```

确保与前端开发服务器的端口一致。

### Q: 找不到 .env 文件

**A:** 创建 `.env` 文件并添加配置：

```bash
cd server
cat > .env << 'EOF'
PORT=3000
NODE_ENV=development
MONGODB_URI=mongodb://localhost:27017/future-you
JWT_SECRET=future-you-secret-key-2024-very-secure
JWT_EXPIRES_IN=7d
CORS_ORIGIN=http://localhost:5173
EOF
```

## 开发建议

### 1. 使用浏览器开发工具

- **Chrome**: F12 打开开发者工具
- **Firefox**: F12 打开开发者工具
- **Safari**: 开发 > 显示 Web 检查器

### 2. 查看 API 日志

后端控制台会显示所有 API 请求：

```
2024-03-17T04:48:00.000Z - GET /api/user/info
2024-03-17T04:48:05.000Z - POST /api/auth/login
```

### 3. 查看 MongoDB 数据

使用 MongoDB Compass 或 mongosh 查看数据：

```bash
# 使用 mongosh
mongosh

# 切换到数据库
use future-you

# 查看所有集合
show collections

# 查询用户数据
db.users.find().pretty()
```

### 4. 重置数据库

在开发过程中，如果需要重置数据库：

```bash
# 使用 mongosh
mongosh

# 删除数据库
use future-you
db.dropDatabase()
```

## 生产环境构建

### 前端构建

```bash
cd client
npm run build
```

构建产物在 `client/dist` 目录

### 后端准备

确保 `server/.env` 中的 `NODE_ENV=production`：

```env
NODE_ENV=production
```

### 使用 PM2 运行生产环境

```bash
cd server

# 安装 PM2（如果未安装）
npm install -g pm2

# 启动应用
pm2 start app.js --name future-you

# 查看状态
pm2 status

# 查看日志
pm2 logs future-you

# 停止应用
pm2 stop future-you

# 重启应用
pm2 restart future-you
```
