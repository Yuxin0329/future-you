## 🛠 技术实现

### 后端技术栈
- **Express.js** - Web框架
- **Multer** - 文件上传处理
- **原生JSON文件存储** - 数据持久化
- **RESTful API** - API设计规范

### 前端技术栈
- **Vue 3 Composition API** - 框架
- **Element Plus** - UI组件库
- **Quill.js** - 富文本编辑器
- **MediaRecorder API** - 浏览器原生录音
- **Vue Router** - 路由管理
- **Pinia** - 状态管理

### 数据流
```
前端组件 → Pinia Store → Axios Request → 后端API → 数据模型 → JSON文件
```

## 📡 API 端点清单

### 情侣结对 API
- `POST /api/partner/apply` - 发起结对申请
- `GET /api/partner/applications/pending` - 获取待处理申请
- `GET /api/partner/applications/sent` - 获取发送的申请
- `POST /api/partner/applications/:applicationId/accept` - 接受申请
- `POST /api/partner/applications/:applicationId/reject` - 拒绝申请
- `GET /api/partner/my` - 获取我的情侣信息
- `POST /api/partner/dissolve` - 解除情侣关系

### 信件 API
- `POST /api/letter` - 创建信件
- `POST /api/letter/upload-audio` - 上传音频文件
- `GET /api/letter/sent` - 获取发送的信件
- `GET /api/letter/received` - 获取收到的信件
- `GET /api/letter/:letterId` - 获取信件详情
- `DELETE /api/letter/:letterId` - 删除信件

## 📝 使用流程

### 情侣结对流程
1. 用户A登录 → 点击"我的伴侣"
2. 输入用户B的用户名 → 点击"发起申请"
3. 用户B登录 → 看到"待处理邀请"
4. 用户B点击"接受" → 结对成功
5. 双方都可以写信给对方
6. 如需解除 → 点击"解除关系"确认

### 写信流程
1. 登录后 → 点击"写信"或"给TA写信"
2. 输入信件标题
3. 选择喜欢的信纸样式
4. 使用富文本编辑器编写内容
5. 可选：录制语音留言
6. 选择送达时间（必须是未来时间）
7. 点击"发送" → 信件将在指定时间发送

## 🔧 已知限制

1. **音频文件大小** - 限制为10MB
2. **录音格式** - 使用audio/webm格式（部分旧浏览器可能不支持）
3. **定时发送** - 目前仅保存为pending状态，需要定时任务执行实际发送
4. **并发限制** - JSON文件存储在高并发场景下可能存在性能问题
5. **数据备份** - 需要手动备份JSON文件
