### 第3步：上传 dist 目录

**重要提示：**
- 不要只上传 `dist/index.html`
- 要上传**整个 `dist/` 目录**（包含所有子文件夹）

**Windows 操作：**
1. 打开文件资源管理器
2. 找到 `dist/` 目录（解压后的位置）
3. 选中 `dist/` 文件夹（不是打开它）
4. 拖拽到 Netlify Drop 的上传区域
5. 松开鼠标，等待上传

**Mac 操作：**
1. 打开 Finder
2. 找到 `dist/` 目录
3. 拖拽到 Netlify Drop 的上传区域
4. 松开鼠标，等待上传

- ⏱️ 总计：**约1-3分钟完成**

**期间你会看到：**

## 🔧 后续配置

### 配置后端 API 地址

**当前情况：**
- 前端部署到了 Netlify（云端）
- 后端还在用 ngrok（本地，不稳定）

**暂时解决方案：**
- 前端 API 地址配置文件：`client/.env.production`
- 目前配置为：ngrok URL（临时）
- 后端改成 Railway/Render 后，更新这个文件

**如何更新 API 地址：**

**方式A：修改 .env.production 文件**
1. 打开 `client/.env.production` 文件
2. 将 `VITE_API_BASE_URL` 改成新的后端地址
3. 重新构建：`npm run build`
4. 重新上传 `dist/` 目录到 Netlify Drop

**方式B：在 Netlify 控制台配置（推荐）**
1. 访问：https://app.netlify.com
2. 登录你的 Netlify 账户（如果没有，注册一个）
3. 找到你的网站
4. 进入 "Site settings" → "Environment variables"
5. 添加环境变量：
   - Key: `VITE_API_BASE_URL`
   - Value: 后端的新地址（比如 Railway 的地址）
6. 保存后，重新触发部署

## 📞 故障排除

### 问题1：上传后找不到 index.html

**原因：** 目录结构不对

**解决方法：**
- 确保上传的是 `dist/` 目录（包含 `index.html`）
- 不要上传 `dist/` 的父目录

### 问题2：部署后网站 404

**原因：** Netlify 的配置问题

**解决方法：**
- 在 Netlify 控制台检查部署日志
- 确保上传了正确的文件
- 重新触发部署

### 问题3：API 请求失败（CORS 或 404）

**原因：** 后端 ngrok 隧道不稳定

**解决方法：**
- 检查后端服务是否运行
- 重新启动 ngrok
- 更新前端 API 地址

### 问题4：CSS 或 JS 文件 404

**原因：** 某些资源没有正确上传

**解决方法：**
- 重新上传 `dist/` 目录
- 清除浏览器缓存（Ctrl + Shift + R）
- 检查控制台是否有错误
